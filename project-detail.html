Space Ship

Unity 6 | C# | Solo Project
Space Ship 2D is a complete 2D space shooter developed as a solo project using Unity 6 and C#. The game includes three playable stages, unique boss encounters, enemy AI, inventory and progression systems, and optimized gameplay through Object Pooling and performance profiling. This project reflects my approach to building maintainable gameplay systems and delivering polished player experiences.
View Source Code Play on Itch.io

Gameplay Features

  • Responsive Player Controls: Smooth spaceship movement with mouse aiming, left-click shooting, and directional dash.
  • Three Playable Stages: Progress through three handcrafted levels with increasing difficulty.
  • Unique Boss Battles: Face three bosses with distinct attack patterns and multi-phase behaviors.
  • Dynamic Enemy AI: Enemies track the player and execute different attack behaviors.
  • Inventory & Item System: Collect items to restore health, recover energy, and enhance combat.
  • Skill-Based Combat: Use multiple ship abilities strategically during battles.
  • Progressive Difficulty: Enemy waves and boss mechanics become more challenging as the game progresses.

Technical Highlights

  • Designed modular gameplay systems with clean, maintainable C# architecture.
  • Developed a flexible inventory system supporting collectible items, consumables, and gameplay progression.
  • Used ScriptableObjects for flexible and data-driven game configuration.
  • Implemented Object Pooling to reduce runtime allocations and improve performance.
  • Profiled CPU and memory usage using Unity Profiler.

Challenges & Solutions

Frequent Instantiate/Destroy

Challenge: Frequent Instantiate and Destroy calls for bullets, enemies, and explosion effects caused unnecessary memory allocations, resulting in GC spikes and unstable frame rates during intense gameplay.

Solution: Implemented an Object Pooling system to reuse bullets, enemies, and visual effects instead of creating and destroying objects repeatedly.

Result: Reduced runtime allocations and garbage collection, resulting in smoother gameplay and more stable FPS during combat.

Excessive UI Updates

Challenge: UI elements such as the inventory and player status were being refreshed every frame, causing unnecessary redraws and reducing rendering performance.

Solution: Combined the Observer and Dirty Flag patterns so UI elements update only when the underlying game data changes.

Result: Eliminated redundant UI redraws, improved rendering efficiency, and maintained cleaner, decoupled UI logic.

Inventory

Challenge: Keeping inventory data synchronized with the UI became difficult as new item types were added.

Solution: Designed a modular inventory system with event-driven updates to keep gameplay logic and UI loosely coupled.
